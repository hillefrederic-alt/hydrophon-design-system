# Design Tokens

Zentrale Quelle fuer alle Gestaltungswerte im Hydrophon Design-System.

## Was sind Design Tokens?

Design Tokens sind benannte Variablen fuer Gestaltungswerte wie Farben, Abstände und Schriftgroessen. Sie ermöglichen konsistente Gestaltung und einfache Anpassungen.

**Beispiel:**
```json
// Token-Definition
{
  "hydrophon": {
    "blau": {
      "500": {
        "$value": "#008BD2",
        "$type": "color"
      }
    }
  }
}
```
```css
/* CSS-Variable (generiert) */
--hydrophon-blau-500: #008BD2;
```

## Token-Architektur

### Drei Ebenen

```
Primitive Tokens    ->    Semantic Tokens    ->    Component Tokens
(Basis-Werte)             (Bedeutung)              (Verwendung)

hydrophon.blau.500  ->    color.action.primary  ->  button.primary.background.default
#008BD2                   (Primaere Aktion)         (Primary Button Hintergrund)
```

**Warum drei Ebenen?**
- **Primitives:** Reine Werte ohne Kontext (Farbskala 50-900)
- **Semantics:** Bedeutung im System (action, error, surface)
- **Components:** Konkrete Anwendung (button, input, modal)

## Token-Dateien

| Datei | Inhalt | Tokens |
|-------|--------|--------|
| `colors.json` | Farbpalette, Produktlinien, Semantische Farben | ~80 |
| `typography.json` | Schriftfamilien, Groessen, Gewichte | ~25 |
| `spacing.json` | Abstaende, Grid, Breakpoints | ~30 |
| `effects.json` | Schatten, Rundungen | ~15 |
| `buttons.json` | Button-Varianten und -Zustaende | ~50 |
| `forms.json` | Formular-Komponenten | ~50 |
| `navigation.json` | Header, Footer, Breadcrumb | ~25 |
| `feedback.json` | Modal, Toast, Tooltip, Loading | ~40 |

**Gesamt:** ~315 Design Tokens

## Token-Format

Wir verwenden das W3C Design Tokens Community Group (DTCG) Format:

```json
{
  "token-name": {
    "$value": "wert",
    "$type": "color|dimension|fontFamily|..."
  }
}
```

**Unterstuetzte Typen:**
- `color` - Farbwerte (#hex, rgb, hsl)
- `dimension` - Groessen (px, rem)
- `fontFamily` - Schriftfamilien
- `fontWeight` - Schriftgewichte
- `number` - Zahlen (z.B. line-height)
- `duration` - Zeitwerte (ms, s)
- `shadow` - Schatteneffekte

## Referenzen (Aliase)

Tokens koennen andere Tokens referenzieren:

```json
{
  "button": {
    "primary": {
      "background": {
        "default": {
          "$value": "{hydrophon.blau.500}",
          "$type": "color"
        }
      }
    }
  }
}
```

Die `{token.pfad}` Syntax erstellt Verknuepfungen. Aendert sich `hydrophon.blau.500`, aktualisieren sich alle Referenzen automatisch.

## Build-Prozess

### Style Dictionary

Token-Kompilierung mit Style Dictionary:

```bash
# Installation (einmalig)
npm install

# Tokens kompilieren
npm run build
```

**Ausgabe:**
```
design-system/build/
├── css/
│   └── variables.css    # CSS Custom Properties
└── json/
    └── tokens.json      # Flaches JSON fuer Tools
```

### Konfiguration

`config.json`:
```json
{
  "source": ["tokens/**/*.json"],
  "platforms": {
    "css": {
      "transformGroup": "css",
      "buildPath": "build/css/",
      "files": [{
        "destination": "variables.css",
        "format": "css/variables",
        "options": {
          "outputReferences": true
        }
      }]
    }
  }
}
```

**outputReferences: true** bedeutet, dass CSS-Variablen Referenzen erhalten:
```css
--button-primary-background-default: var(--hydrophon-blau-500);
```

## Verwendung

### CSS importieren

```css
@import 'design-system/build/css/variables.css';

.meine-komponente {
  background-color: var(--color-surface-primary);
  padding: var(--spacing-4);
  border-radius: var(--border-radius-md);
}
```

### JavaScript/TypeScript

```javascript
// JSON-Tokens fuer Runtime-Zugriff
import tokens from 'design-system/build/json/tokens.json';

const primaryColor = tokens['hydrophon-blau-500'];
```

# Grid & Breakpoints

Responsive Layout-System fuer konsistente Gestaltung.

## Grid-System

12-Spalten-Grid fuer maximale Flexibilitaet.

### Spaltenaufteilung

| Spalten | Verwendung |
|---------|------------|
| 12 | Full Width |
| 6+6 | Zweispaltig |
| 4+4+4 | Dreispaltig |
| 3+3+3+3 | Vierspaltig |
| 8+4 oder 4+8 | Asymmetrisch |

### CSS Grid Implementierung

```css
.container {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: var(--grid-gutter);
  max-width: var(--grid-max-width);
  margin: 0 auto;
  padding: 0 var(--grid-margin);
}

/* Spannende Spalten */
.col-6 { grid-column: span 6; }
.col-4 { grid-column: span 4; }
.col-3 { grid-column: span 3; }
```

### Grid-Variablen

| Variable | Mobile | Tablet | Desktop |
|----------|--------|--------|---------|
| `--grid-columns` | 4 | 8 | 12 |
| `--grid-gutter` | 16px | 24px | 32px |
| `--grid-margin` | 16px | 24px | 32px |
| `--grid-max-width` | - | - | 1280px |

## Breakpoints

### Definitionen

| Name | Variable | Wert | Geraete |
|------|----------|------|---------|
| Mobile | `--breakpoint-sm` | 640px | Smartphones |
| Tablet | `--breakpoint-md` | 768px | Tablets Portrait |
| Desktop | `--breakpoint-lg` | 1024px | Tablets Landscape, Laptops |
| Large | `--breakpoint-xl` | 1280px | Desktop |
| Extra Large | `--breakpoint-2xl` | 1536px | Grosse Monitore |

### Mobile-First Ansatz

Media Queries von klein nach gross:

```css
/* Mobile (Standard) */
.element { font-size: 14px; }

/* Tablet und groesser */
@media (min-width: 768px) {
  .element { font-size: 16px; }
}

/* Desktop und groesser */
@media (min-width: 1024px) {
  .element { font-size: 18px; }
}
```

### Breakpoint-Variablen in CSS

```css
/* Breakpoints als Custom Properties */
:root {
  --breakpoint-sm: 640px;
  --breakpoint-md: 768px;
  --breakpoint-lg: 1024px;
  --breakpoint-xl: 1280px;
  --breakpoint-2xl: 1536px;
}

/* HINWEIS: CSS Media Queries koennen keine var() nutzen */
/* Breakpoint-Werte direkt in Media Queries verwenden */
@media (min-width: 768px) { /* --breakpoint-md */ }
```

## Responsive Patterns

### Container

```css
.container {
  width: 100%;
  max-width: var(--grid-max-width);
  margin: 0 auto;
  padding-left: var(--grid-margin);
  padding-right: var(--grid-margin);
}
```

### Responsive Grid mit auto-fit

Automatische Spaltenanzahl basierend auf Viewport:

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(
    auto-fit,
    minmax(min(280px, 100%), 1fr)
  );
  gap: var(--spacing-6);
}
```

### Touch Targets

WCAG 2.1 AA erfordert 44x44px Touch Targets:

| Variable | Wert | Verwendung |
|----------|------|------------|
| `--touch-target-min` | 44px | WCAG AA Minimum |
| `--touch-target-compact` | 32px | Nur Desktop |

## Spacing-Anpassungen

### Responsive Spacing

| Spacing | Mobile | Desktop |
|---------|--------|---------|
| Section Padding | 48px (spacing-12) | 96px (spacing-24) |
| Component Gap | 16px (spacing-4) | 24px (spacing-6) |
| Card Padding | 16px (spacing-4) | 24px (spacing-6) |

### Beispiel

```css
.section {
  padding: var(--spacing-12) 0;
}

@media (min-width: 1024px) {
  .section {
    padding: var(--spacing-24) 0;
  }
}
```
