## Grid-Layout
### CSS Grid auto-fit Pattern
- grid-template-columns: repeat(auto-fit, minmax(min(280px, 100%), 1fr))
- gap: var(--card-gap)
- Keine Media Queries nötig!
- Erklärung warum dieses Pattern funktioniert

### Responsive Verhalten
- < 560px: 1 Spalte
- 560-840px: 2 Spalten
- 840-1120px: 3 Spalten
- > 1120px: 4 Spalten
- Automatisch durch minmax()

## Hover-Effekt
- Transform: translateY(-4px) - GPU-beschleunigt
- Shadow: sm → md
- Transition: 300ms ease
- prefers-reduced-motion: Keine Animation

## Performance
- will-change: transform (nur auf Hover-Kandidaten)
- GPU-beschleunigt: nur transform und opacity animieren
- Lazy Loading: loading="lazy" auf Images

## Tokens
- Token-Tabelle

## Code-Beispiele
### HTML
- Vollständige Card mit article
- a-Element wrapping für gesamte Card klickbar
- Specs als ul

### CSS
- Grid-Container
- Card-Styling
- Hover-Animation
- Reduced-Motion

### Performance-optimierte Shadow-Animation
- ::after Pseudo-Element mit opacity
- Nicht box-shadow direkt animieren

## Barrierefreiheit
- article für semantische Gruppierung
- Heading-Hierarchie (h3 in Grid)
- :focus-within für Card-Focus
- Lazy Loading Attribut

## Best Practices / Nicht verwenden
- Best Practices: Auto-fit Grid, GPU-Animation, Lazy Loading

**data-table.md:**

## Übersicht
- Zweck: Produktvergleiche, technische Spezifikationen
- B2B-Kontext: Hydrophon Produktdaten (Breite, Höhe, Material, Preis)

## Anatomie
- table-Element
- caption (Tabellentitel, für Screen Reader)
- thead mit th scope="col"
- tbody mit th scope="row" für Zeilentitel
- Optional: tfoot für Summen

## Header-Struktur
### Spaltenheader (thead)
- scope="col" auf allen th
- Styling: Hintergrund neutral.100, 2px Border unten

### Zeilenheader (erste Spalte)
- scope="row" auf th in tbody
- Produktname als Zeilenheader

## Zebra-Striping
### Subtiler Kontrast
- tr:nth-child(even): neutral.50
- tr:nth-child(odd): transparent
- Max 1.5:1 Kontrast zwischen Zeilen
- Text bleibt 4.5:1 auf beiden Hintergründen

### Warum subtil?
- Starke Stripes = visuelle Störung
- Leichte Stripes = verbesserte Lesbarkeit

## Row Hover
- Zusätzlich zu Stripes
- primary.50 bei Hover
- Transition für smooth

## Responsive-Verhalten
### Horizontaler Scroll (empfohlen)
- .table-container: overflow-x: auto
- table: min-width: 600px
- Scroll-Indikatoren (Shadows am Rand)
- Behält Semantik für Screen Reader

### Alternative: Stacked Cards
- Nur für einfache Tabellen
- thead versteckt (visually-hidden)
- td::before mit data-label
- Komplexer, aber mobil-freundlicher

## Caption
- Beschreibt Tabelleninhalt
- Für Screen Reader wichtig
- Styling wie normale Heading

## Tokens
- Token-Tabelle aus tables.json

## Code-Beispiele
### Basis-Tabelle
- Vollständige HTML-Struktur
- scope-Attribute
- caption

### CSS
- Basis-Styling
- Zebra-Stripes
- Hover
- Responsive Container

### Responsive mit Scroll-Indikatoren
- CSS Shadows für Scroll-Hinweis

## Barrierefreiheit
- WCAG 1.3.1: Info and Relationships (scope!)
- caption für Tabellenbeschreibung
- Keine merged cells ohne headers-Attribut
- Keyboard: Tab durch interaktive Elemente

## Best Practices / Nicht verwenden
- Best Practices: Immer scope, caption, subtile Stripes
- Nicht verwenden: div-Tabellen, starke Zebra-Kontraste, fehlende Header
